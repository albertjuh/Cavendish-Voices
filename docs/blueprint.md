# **App Name**: Cavendish Voices

## Core Features:

- Landing & Navigation: A welcome page with Cavendish University branding, mission statement, and a clear navigation menu to access various sections of the application.
- Suggestion Submission Form: A comprehensive form allowing students to submit anonymous or identified suggestions, complaints, recommendations, and appreciations, with fields for student details (optional), department, category, title, message, and priority level. Includes client-side validation.
- Live Suggestion Display: A page that displays recently submitted suggestions in a card-based layout, allowing users to search, filter by category or department, and sort by date. (Note: For the MVP, suggestions are simulated or stored client-side for demonstration purposes only, not persistently stored on a server).
- Admin Dashboard Demo: A frontend-only interface demonstrating administrative capabilities, showing mock statistics (total, pending, resolved suggestions, category statistics) and a table of submissions. Includes UI controls to simulate actions like 'Mark Resolved', 'Review', and 'Delete' (these actions will only affect the current client-side view for demonstration).
- Dynamic UI Feedback: Provides immediate visual confirmation upon successful submission and implements form validation to guide users.
- Responsive Design Adaptability: Ensures the entire application interface fluidly adapts and maintains usability across a wide range of devices, from mobile phones to desktop monitors.

## Style Guidelines:

- Primary Color: A sophisticated, deep university blue (#1F509F) to evoke professionalism and trust, suitable for main interactive elements and branding.
- Background Color: A very light, desaturated blue (#F2F5FA) that subtly reinforces the brand without being overpowering, ensuring content remains highly readable.
- Accent Color: A vibrant yet elegant gold (#E4B32E) to highlight calls to action, important notices, and interactive elements, providing a complementary pop against the primary blue.
- Headlines: 'PT Sans' (humanist sans-serif) for its modern yet approachable character, perfect for engaging titles and dashboard labels.
- Body Text: 'Inter' (grotesque sans-serif) chosen for its exceptional readability and neutral aesthetic, ensuring long suggestion messages and descriptions are clear.
- Utilize a consistent set of line-style icons with rounded corners, complementing the modern and clean aesthetic, for navigation, form actions, and data representation in the dashboard.
- Implement a flexible grid-based layout for dashboard widgets and a card-based structure for suggestion listings, ensuring responsiveness across all devices. Employ soft shadows and slightly rounded corners for elements to create a modern, approachable feel.
- Incorporate subtle hover effects on interactive elements and smooth transitions for state changes (e.g., suggestion filtering, form submission feedback) to enhance user experience without distraction. An animated hero section on the landing page will create an engaging first impression.
